# yo-s5 产品需求规格

> 本文档面向 AI 开发助手，描述 yo-s5 的产品定位、核心场景和设计决策。

---

## 1. 产品定位

**yo-s5 是一个一键部署 SOCKS5 代理的工具。**

目标用户是需要在 Linux 服务器上快速搭建 SOCKS5 代理的开发者。传统方式需要手动安装软件、配置认证、开放端口、测试连通性，yo-s5 把这些全部自动化。

核心理念：**一条命令，代理就能用。**

---

## 2. 核心场景

### 2.1 全自动部署

**需求：** 用户拿到一台新的 Linux 服务器，想尽快搭一个带认证的 SOCKS5 代理。

**行为：**
- 用户运行 `yo-s5`
- 系统检测 Docker 是否安装，未安装则自动安装
- 系统启动 Docker 服务
- 自动生成随机密码
- 自动查找可用端口
- 拉取 GOST 镜像，启动容器
- 获取服务器公网 IP
- 测试代理连通性
- 输出完整的代理配置信息（JSON 格式），包含 IP、端口、用户名、密码

### 2.2 交互式部署

**需求：** 用户想自定义端口和密码，而不是用自动生成的。

**行为：**
- 用户运行 `yo-s5 -i`
- 交互式提示输入密码（或回车使用随机生成的）
- 交互式提示输入端口（或回车使用自动选择的）
- 后续流程同自动模式

### 2.3 重复运行

**需求：** 用户已经跑过一次，想重新部署（换端口/换密码）。

**行为：**
- 自动清理同名旧容器（停止 + 删除）
- 重新走完整部署流程
- 不影响服务器上的其他 Docker 容器

---

## 3. 设计决策

| 决策 | 理由 |
|------|------|
| 基于 Docker + GOST | 无需编译安装，GOST 是成熟的代理工具，Docker 确保环境隔离 |
| 自动安装 Docker | 降低使用门槛，用户不需要预先准备环境 |
| 随机密码 + 随机端口 | 自动模式追求零配置，安全性由随机性保证 |
| JSON 格式输出配置 | 方便程序化使用，可直接粘贴到客户端配置 |
| 连通性测试 | 部署完自动验证，确保代理确实可用 |
| 拒绝 WSL 环境 | WSL 网络栈不完整，SOCKS5 代理在 WSL 中无法正常对外服务 |

---

## 4. 当前已知限制

- **仅 Linux 支持：** 依赖 Docker 和 Linux 网络栈
- **WSL 不可用：** 主动检测并拒绝在 WSL 中运行
- **固定用户名：** 认证用户名固定为特定值，仅密码可自定义
- **单容器模式：** 每次运行只部署一个代理实例
