# yo-auto 产品需求规格

> 本文档面向 AI 开发助手，描述 yo-auto 的产品定位、核心场景和设计决策。
> 开发时应以本文档为需求依据，代码实现服务于这里描述的用户场景。

---

## 1. 产品定位

**yo-auto 是一个运行在 Windows 上的个人自律与生活助手。**

目标用户是长时间在电脑前工作的开发者。他们容易忘记时间、忽略健康、沉迷工作或娱乐。yo-auto 通过语音提醒、屏幕锁定、密码强制等手段，帮助用户建立健康的作息节奏。

核心理念：**电脑应该主动关心你，而不是等你想起来。**

它不是一个通用的任务调度器——它是一个"说人话"的、有温度的桌面伴侣。整点报时用真实钟声，语音提醒用自然的中文 TTS，节日和节气会被感知到。

---

## 2. 核心场景

### 2.1 定时语音提醒

**需求：** 用户希望在工作时段内，每隔一段时间收到语音提醒，内容可以自定义。

典型场景：
- 每 30 分钟提醒喝水
- 每 60 分钟提醒站起来活动
- 午饭时间提醒吃饭
- 下班时间提醒收工

**行为：**
- 用户通过 `.rhai` 脚本定义时间范围、间隔、提醒内容
- 语音通过 TTS 合成后直接播放，不弹窗、不打断工作流
- TTS 音频会被缓存（按文本内容哈希），相同内容不重复请求
- 系统会提前 1 分钟预热即将播放的 TTS，确保播放无延迟

### 2.2 整点报时

**需求：** 用户希望像老式钟一样，每到整点能听到钟声，帮助感知时间流逝。

**行为：**
- 整点播放嵌入的钟声音效
- 可通过脚本配合 TTS 播报当前时间（如"现在是下午三点"）
- 钟声和语音可以组合使用

### 2.3 锁屏感知

**需求：** 用户锁屏意味着离开了，不应该对着空气说话；用户解锁意味着回来了，可以说一句欢迎回来。

**行为：**
- 系统通过 Windows 会话事件实时监听锁屏/解锁
- 锁屏时触发 `on_lock` 钩子——脚本可以选择暂停提醒、记录离开时间等
- 解锁时触发 `on_unlock` 钩子——脚本可以播报离开时长、待办事项等
- 用户锁屏期间，定时 `on_tick` 仍然执行，但脚本可通过 `screen_locked()` 判断是否跳过语音

### 2.4 强制锁屏与密码管控

**需求：** 用户希望在特定时间段（如深夜）强制自己离开电脑，即使自控力不足也能生效。

**行为：**
- 脚本可调用 `lock_screen()` 锁定屏幕
- 脚本可调用 `change_password()` 将密码改为一个难以手动输入的字符串，从而阻止用户轻易解锁
- 到了允许使用的时间，脚本调用 `restore_password()` 恢复原密码
- **自愈机制：** 如果程序崩溃或被意外终止，下次启动时自动检测并恢复密码，避免用户被锁死

### 2.5 天气播报

**需求：** 用户早上想知道今天天气怎么样，不用自己去查。

**行为：**
- 脚本通过 `get_weather("城市名")` 获取实时天气
- 返回天气状况、温度、体感温度、湿度、风向风力
- 脚本自行组织播报内容（如"今天北京晴，23度，微风"）

### 2.6 节日与节气感知

**需求：** 中国人的时间感和传统节日、节气紧密相连。系统应该知道今天是什么日子。

**行为：**
- `get_today_festival()` — 检测今天是否是节日（春节、元宵、端午、中秋等，涵盖农历和公历节日）
- `get_today_solar_term()` — 检测今天是否是二十四节气
- `get_today_special()` — 优先返回节日，其次节气
- `days_until_spring_festival()` — 距离下一个春节还有几天
- 脚本可利用这些信息定制语音内容（如"今天是冬至，记得吃饺子"）

### 2.7 Web 管理界面

**需求：** 用户希望通过浏览器管理规则脚本，而不是手动编辑文件。

**行为：**
- 启动时可选 `--web` 参数开启 Web UI（默认端口 9999）
- 提供嵌入式 Vue.js 单页应用，无需额外部署
- 功能：
  - 查看所有已加载规则的状态
  - 在线编辑、创建、删除、重命名 `.rhai` 脚本
  - 模拟执行脚本——指定时间范围，预览会产生哪些语音/报时事件
  - 导入导出脚本（单个或全部）
  - 管理日历事件（可视化的时间块）
  - 管理环境变量配置（TTS API Key、天气密钥等）
  - 热重载规则（无需重启服务）

### 2.8 开机自启动

**需求：** 用户不想每次开机都手动启动，希望一条命令搞定自启。

**行为：**
- `yo run auto --autostart install` — 在 Windows 启动目录创建 VBS 脚本，通过 Git Bash 启动 yo-auto
- `yo run auto --autostart remove` — 移除自启
- `yo run auto --autostart status` — 查看当前状态
- 依赖 Git Bash 环境（自动检测路径）

## 3. 规则脚本系统

### 3.1 设计理念

用户通过 Rhai 脚本描述"什么时候做什么事"。脚本放在 `~/.yo/rules/` 目录下，每个 `.rhai` 文件是一个独立的规则。

选择 Rhai 的原因：
- 语法接近 JavaScript/Rust，学习成本低
- 纯 Rust 实现，无外部依赖，嵌入简单
- 沙箱安全，脚本无法直接访问文件系统或网络

### 3.2 触发器模型

每个脚本通过返回配置对象的方式声明触发条件：

| 字段 | 含义 |
|------|------|
| `time_range` | 生效时间范围，如 `"08:00-22:00"`，支持跨午夜如 `"22:00-06:00"` |
| `interval_minutes` | 在时间范围内每隔多少分钟触发一次 `on_tick` |
| `weekdays` | 生效的星期，如 `[1,2,3,4,5]` 表示工作日。空数组表示每天 |
| `events` | 监听的事件列表：`"unlock"`（解锁）、`"lock"`（锁屏） |
| `enabled` | 是否启用此规则 |

### 3.3 生命周期钩子

| 钩子 | 触发时机 | 典型用途 |
|------|----------|----------|
| `on_mount(state)` | 进入时间范围时 | 初始化状态、播报开始工作 |
| `on_tick(state)` | 每个 interval 到达时 | 定时提醒、整点报时 |
| `on_unlock(state)` | 用户解锁屏幕时 | 欢迎回来、播报离开时长 |
| `on_lock(state)` | 用户锁定屏幕时 | 记录离开时间 |
| `on_destroy(state)` | 离开时间范围时 | 清理状态、播报结束 |

每个钩子接收一个 `state` 对象（Rhai Map），脚本可在其中存取任意键值。状态在同一脚本的不同钩子调用之间持久化（持久化到磁盘），跨重启保留。

### 3.4 可用 API

**时间类：**
- `hour()`, `minute()`, `second()` — 当前时间
- `weekday()` — 星期几（1-7）
- `is_weekend()`, `is_workday()`
- `time_str()`, `date_str()` — 格式化时间/日期字符串
- `in_time_range(start, end)` — 当前是否在指定时间范围内
- `weekday_name()` — 中文星期名

**语音类：**
- `speak(text)` — TTS 合成并播放
- `speak(text, pause_ms)` — 播放后额外等待指定毫秒
- `chime(hour)` — 播放整点钟声

**系统类：**
- `screen_locked()` — 当前是否锁屏
- `lock_screen()` — 锁定屏幕
- `enter_sleep()` — 进入睡眠模式
- `shutdown(delay_seconds)` — 延迟关机
- `change_password()` — 修改密码为锁定密码
- `restore_password()` — 恢复原始密码
- `log(message)` — 输出日志

**环境类：**
- `get_env(name)` — 获取配置的环境变量
- `has_env(name)` — 检查环境变量是否存在
- `get_weather(location)` — 获取实时天气

**日历类：**
- `get_today_festival()` — 今日节日
- `get_today_solar_term()` — 今日节气
- `get_today_special()` — 今日特殊日（优先节日）
- `days_until_spring_festival()` — 距春节天数

**脚本间协作：**
- `generate_script_events(script_name)` — 模拟另一个脚本的执行，获取其事件列表
- `configure_tts(api_key, voice)` — 动态配置 TTS 参数

## 4. 平台约束与设计决策

| 决策 | 理由 |
|------|------|
| Windows 优先 | 目标用户是 Windows 桌面开发者，锁屏检测、密码管理、音频播放都依赖 Win32 API |
| 火山引擎 TTS | 中文语音质量好，支持多种音色，API 简单 |
| SHA256 缓存 | 相同文本+音色只请求一次，节省 API 调用，消除播放延迟 |
| TTS 预热 | 提前 1 分钟合成下一轮要播放的内容，确保到时间时立即播放 |
| Git Bash 自启 | Windows 原生启动目录 + VBS 脚本 + Git Bash 是最可靠的方案 |
| 嵌入式 Web UI | 单个二进制文件即可运行，无需安装 Node.js 或部署前端 |
| 单实例锁 | PID 文件保证同时只有一个 yo-auto 运行，避免重复播报 |

## 5. 非功能需求

### 5.1 可靠性
- **密码自愈：** 启动时检查密码状态标记文件，若上次异常退出导致密码未恢复，自动恢复
- **Ctrl+C 处理：** 注册信号处理器，退出前恢复密码
- **单实例保证：** 通过 PID 文件互斥，检测旧进程是否存活

### 5.2 用户体验
- **零延迟播放：** TTS 缓存 + 提前预热，用户感受不到网络延迟
- **静默运行：** 无弹窗、无托盘图标，只通过声音和屏幕锁定与用户交互
- **热重载：** 每小时自动重载规则，Web UI 也可手动触发，无需重启

### 5.3 数据存储
所有用户数据存放在 `~/.yo/` 目录下：
- `config.json` — 全局配置（TTS API Key 等环境变量）
- `rules/*.rhai` — 用户编写的规则脚本
- `rules/states/*.json` — 脚本持久化状态
- `voice/cache/*.mp3` — TTS 音频缓存
- `voice/clock/` — 钟声音效
- `events.json` — 日历事件
- `yo-auto.pid` — 实例锁文件
- `password_changed` — 密码状态标记

---

## 6. 当前已知限制

- **仅 Windows 完整支持：** 锁屏检测、密码管理、音频播放在 Linux/macOS 上为 stub 或有限实现
- **TTS 依赖外部 API：** 无网络时语音功能不可用（但缓存的内容仍可播放）
- **密码管理仅支持本地账户：** 域账户或 Microsoft 账户可能不适用
- **Web UI 无认证：** 仅监听本地端口，适合个人使用，不应暴露到公网
- **日历事件与脚本未深度集成：** 日历事件目前通过 Web UI 独立管理，脚本无法直接查询日历事件
